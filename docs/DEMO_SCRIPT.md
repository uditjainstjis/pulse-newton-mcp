# Pulse Demo Script

## 60-second version

1. Open Pulse home screen.
   Say: "Pulse is a Newton MCP-powered student operating system. It does not just show data. It tells a student what matters next."

2. Point at the current source badge.
   Say: "On my machine, Pulse uses live Newton data. In hosted review mode, it falls back to scenario-based demos so anyone can test the planning engine."

3. Switch between `Founder mode`, `Deadline spiral`, and `Comeback week`.
   Say: "These are not cosmetic states. The health score, risk pressure, backlog, and intervention sequence all change."

4. Open the dynamic planning section.
   Change duration from `30` to `120`.
   Say: "This is the core interaction. A student rarely has a perfect study block. Pulse adapts the plan to the time they actually have."

5. Show the recovery engine and five-day forecast.
   Say: "It predicts where the week becomes fragile and converts backlog into a practical recovery sequence."

6. Close on Newton integration.
   Say: "The data contract is aligned to Newton MCP surfaces like courses, schedule, lectures, assignments, QOTD, and arena stats."

## 90-second version

### Opening

"Students do not need more dashboards. They need a system that tells them what to do next. Pulse uses Newton MCP data to produce a daily brief, risk score, workload forecast, and recovery plan."

### Show reviewer mode

"A reviewer opening the hosted site may not have a Newton login. So Pulse includes scenario switching that lets them test how the planner behaves across different student realities."

### Switch to deadline spiral

"Here the backlog grows, deadlines compress, weak-topic drag spikes, and the planner becomes more urgency-heavy."

### Switch to comeback week

"Here the same product becomes more momentum-oriented because the student is already recovering and should protect consistency rather than panic."

### Show the intervention blueprint

"This is the part I believe matters most. If I only have 30 minutes, I should not get the same answer as if I have 3 hours. Pulse changes the sequence accordingly."

### Close

"This is useful because Newton already has the academic signals. Pulse turns them into decisions."

## Reviewer talking points

- "This is built on real Newton account data locally."
- "The hosted review mode exists so judges can test product behavior without authentication friction."
- "The value is not reporting. The value is prioritization."
- "The strongest differentiator is the time-aware intervention planner."
